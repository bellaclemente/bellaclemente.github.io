Goals
The development outsourcing company Lazaros, had a few identified pain points on their website (lazaros.com.br), to improve usability and access to information. The main goals for this UX/UI project were:
Analyze the website, web, and mobile, concerning UX, usability, and metrics;
Map improvements according to their degree of severity/urgency;
Document improvements by text and/or adjustments in Figma.
Accessibility analysis
To map out the accessibility status of the system, the guidelines from WCAG (Web Content Accessibility Guidelines) in its 2.1 version were considered. Criteria for physical and cognitive disabilities were analyzed and improvements were documented.

‚àô  WCAG checklist
An extensive checklist comprehending different guidelines from WCAG, checking aspects such as availability to vision, hearing, and touch; controls and navigation; user comprehension of the system; and assistive technologies availability.

‚àô  Color contrast
Main screens and components were checked for specific color contrast to understand and suggest new color options for those that didn't match the necessary color contrast (minimum of 4.5:1, WCAG AA level).

‚àô  Color blindness 
Different pages were tested with color blindness filters to understand when UI elements  don't have enough contrast between background and foreground information.

Flows and components evaluation
The website and its components were evaluated, on desktop and mobile platforms, aiming for ease of access to main information. Considering interaction and accessibility principles, visuals and CSS were looked at in detail to find pain points that could be improved.
Heuristic evaluation
Using Jacobs Nielsen's 10 Usability Heuristics for User Interface Design, a Heuristic Evaluation has been carried out. Analyzing different aspects of the system, such as visibility of system status, control and user freedom, error prevention, consistency and patterns, and others. 
The overall website corresponded only partially to the heuristics.

UX metrics
Both Google Analytics and Lighthouse reports were studied, identifying metrics that could be improved to speed up the website loading process, retain more users during navigation, and see which content is more accessed by users.

‚àô  Google Analytics
Google Analytics metrics were analyzed throughout time, comparing different metrics and establishing connections between different data, understanding which content performs best and current user retention.

‚àô  Lighthouse reports
Using the Lighthouse tool, mobile and web versions were tested to evaluate system performance, accessibility, and SEO (Search Engine Optimization). 

Results
The improvements were listed in a written document, detailing them, identifying in which analyses they were found and ranking them in order of severity.
The improvement suggestions that indicated major changes to the interface were prototyped in Figma to detail the specifications for development.
